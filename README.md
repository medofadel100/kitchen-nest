# KitchenNest — نظام تصميم وتسعير المطابخ لورش النجارة

## اللي اشتغل دلوقتي (Core Logic - تم اختباره فعليًا)

1. **`src/types/index.ts`** — كل الـ types: خامات، وحدات مطبخ، أجهزة، مشروع، تسعير، nesting
2. **`src/data/materials.ts`** — خامات افتراضية (MDF, HPL, أكريليك, ميلامين) بأسعار **placeholder** واضحة (`isPricePlaceholder: true`) — لازم تتعدل بأسعار حقيقية من الموردين
3. **`src/lib/cuttingList.ts`** — بيحول أي وحدة مطبخ (base/wall/tall/corner/drawer) لقايمة قطع خشب حقيقية (جانبين، قاعدة، سقف، ظهر، رف، أبواب، واجهات أدراج)
4. **`src/lib/nesting.ts`** — خوارزمية Guillotine Bin Packing بتوزع القطع على الألواح بأقل هالك، وبتحسب سُمك شفرة المنشار (kerf) بين كل قطعتين
5. **`src/lib/pricing.ts`** — بيربط الاتنين: بياخد الوحدات → يطلع القطع → يعمل nesting فعلي → يحسب **عدد الألواح الحقيقي** (مش المساحة النظرية) → يضيف edge banding + إكسسوارات + هامش ربح → السعر النهائي

جرّبته على مطبخ تجريبي (7 وحدات) وطلع:
```
عدد الألواح المطلوبة: { mdf مقاوم رطوبة: 4 ألواح, mdf عادي: 2 لوح }
السعر النهائي: 16,573 جنيه (شامل هامش ربح 25%)
```

شغّل الاختبار بنفسك: `npm install && npx tsx scripts/test-core-logic.ts`

## اللي لسه ناقص (الخطوات الجاية بالترتيب المنطقي)

### 1. واجهة الرسم (Canvas)
- صفحة Next.js فيها canvas 2D (ينفع React + SVG أو Konva.js) لرسم حدود المطبخ ووحدات قابلة للسحب/التكبير
- Snap تلقائي على `STANDARD_WIDTHS_MM` الموجودة في types
- عرض الـ fixtures (باب/شباك/صرف/مواسير) كعلامات على الحيطة

### 2. عرض نتيجة الـ Nesting بصريًا
- كل `NestingSheetResult` فيه إحداثيات كل قطعة (`xMm`, `yMm`) — سهل ترسمها كـ SVG rectangles فوق مستطيل اللوح، ده هيبقى "cutting sheet" جاهز للطباعة للمشغل

### 3. لوحة تحكم الخامات (Admin)
- CRUD بسيط فوق `Material[]` — كل ورشة تدخل موردينها وأسعارها الحقيقية بدل الـ placeholder
- زرار واضح "الأسعار دي تقديرية - عدّلها من هنا" في أول استخدام

### 4. قاعدة البيانات
زي مشاريعك التانية (Ramon, Romad Laser): **Firebase/Firestore + Vercel** هو الأنسب هنا كمان — نفس الـ pattern، تكامل سريع مع Next.js، ومناسب لعدد الورش والمشاريع المتوقع.
Collections مقترحة: `workshops`, `materials` (per workshop), `projects`, `unitTemplates`

### 5. موديلات 3D للأجهزة
- `Appliance.modelUrl` جاهز في الـ types لمسار glb/gltf
- استخدام `<model-viewer>` أو three.js لعرضها، مع قفل الأبعاد على `standardCutoutMm` عشان لو المستخدم كبّر/صغّر الجهاز يفضل قابل للتركيب فعليًا

### 6. Output للطباعة/PDF
- عرض سعر للعميل (بدون تفاصيل التكلفة الداخلية) + cutting sheet داخلي للورشة
